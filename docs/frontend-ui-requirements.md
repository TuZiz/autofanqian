# 前端 UI 提供模板

这份文档用于约束你后续给我的前端页面说明。

因为你已经明确要求：

- 我不主动实现任何前端 UI
- 只有在你明确要求“做页面”时，我才落对应 UI

所以后续如果你要我接某个页面，请按下面模板给信息。你不一定要全部填满，但越完整，我越不会反复问你。

## 你需要提供什么

### 1. 页面基础信息

- 页面名称
- 路由地址
- 页面用途
- 页面访问权限

示例：

```md
页面名称：开始创作
路由：/dashboard/create
用途：用户输入创意并生成小说大纲
权限：登录后可访问
```

### 2. 页面结构

请直接写页面由哪些区域组成。

示例：

```md
页面结构：
1. 顶部标题区
2. 三步进度条
3. 左侧表单区
4. 右侧模板推荐区
5. 底部操作按钮区
```

### 3. 页面字段

请把每个字段写清楚：

- 字段名
- 类型
- 是否必填
- 占位文案
- 默认值
- 校验规则

示例：

```md
字段：
- 小说类型：单选，必填，无默认值
- 创意描述：多行文本，必填，最多 2000 字
- 目标平台：下拉选择，非必填，默认空
- 目标字数：下拉选择，必填，默认 100 万字
```

### 4. 交互行为

请写清楚页面上每个按钮、切换项、弹窗的行为。

示例：

```md
交互：
- 点击“AI 生成创意”后，用当前小说类型自动填充创意描述
- 点击“生成大纲”后，调用后端接口
- 点击模板卡片后，把模板内容写入创意描述输入框
```

### 5. 状态说明

请明确页面需要哪些状态。

常见状态包括：

- 默认态
- 加载态
- 提交中
- 空状态
- 错误态
- 成功态
- 无权限态

示例：

```md
状态：
- 未选择小说类型时，右侧模板区显示空状态
- 提交中时，生成按钮禁用并显示“生成中”
- 接口失败时，顶部显示错误提示
```

### 6. 数据来源

请告诉我页面数据从哪里来。

可选来源：

- 写死静态数据
- 本地状态
- 调后端接口
- URL 参数
- 登录用户信息

示例：

```md
数据来源：
- 当前用户信息来自 GET /api/auth/session
- 模板数据先写死在前端
- 小说类型先写死在前端
```

### 7. 文案要求

如果你不想让我自行补文案，需要你明确告诉我：

- 用中文还是英文
- 按钮文案
- 标题文案
- 错误提示文案
- 占位提示文案

示例：

```md
文案要求：
- 全中文
- 风格偏产品化，不要太营销
- 按钮文案用“生成大纲”，不要用“开始创作”
```

### 8. 视觉参考

如果你有截图、草图、Figma、竞品图，直接发即可。

你至少告诉我其中一种：

- 像哪张图
- 像哪个网站
- 用什么布局
- 偏什么风格

示例：

```md
视觉参考：
- 顶部像 Linear 的工作台
- 内容区像 Notion 的表单页面
- 不要玻璃风，不要大面积渐变
```

## 最简提交模板

```md
页面名称：
路由：
用途：
页面结构：
字段：
交互：
状态：
数据来源：
文案要求：

```

## 我会怎么用这份文档

你给我页面说明后，我默认只做这两类事之一：

1. 你没明确说“直接做页面”
只输出前端实现说明、组件拆分建议、接口对接方式、状态清单

2. 你明确说“直接做页面”
我才会按你给的说明去实现 UI
