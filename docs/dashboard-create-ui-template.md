## 建议你重点补充的内容

`/dashboard/create` 这类页面，最容易遗漏的是下面这些点：

- 顶部是否有步骤条
- 左右布局还是单栏布局
- 小说类型是否需要卡片选择
- 创意输入是纯文本还是带 AI 辅助
- 模板区是否跟随类型切换
- 生成按钮点击后是直接请求接口，还是先本地校验
- 生成成功后去哪里
- 是否保留草稿

## 详细填写模板

### 1. 页面基础信息

```md
页面名称：
路由：/dashboard/create
页面用途：
访问权限：登录后可访问 / 仅指定角色可访问 / 其他
```

示例：

```md
页面名称：开始创作
路由：/dashboard/create
页面用途：用户填写小说创意并生成大纲
访问权限：登录后可访问
```

### 2. 页面结构

请直接写出页面有哪些区域。

```md
页面结构：
1. 顶部标题区
2. 步骤条
3. 左侧创意表单区
4. 右侧模板推荐区
5. 底部操作区
```

### 3. 字段定义

这个页面通常会包含这些字段，你按需保留或删掉：

```md
字段：
- 小说类型
  类型：单选 / 卡片选择
  必填：是
  默认值：无
  占位文案：无
  校验规则：必须选择一个类型

- 创意描述
  类型：多行文本
  必填：是
  默认值：空
  占位文案：例如主角设定、故事冲突、写作风格
  校验规则：最多 2000 字

- 目标平台
  类型：下拉选择
  必填：否
  默认值：空
  占位文案：选择目标发布平台
  校验规则：无

- 仿书 DNA
  类型：下拉选择
  必填：否
  默认值：空
  占位文案：选择仿书 DNA 风格
  校验规则：无

- 目标字数
  类型：下拉选择
  必填：是 / 否
  默认值：
  占位文案：选择目标字数
  校验规则：
```

### 4. 交互说明

这里是最关键的部分。请明确每个按钮和区域怎么动。

```md
交互：
- 点击某个小说类型后，右侧模板区切换成对应模板
- 点击模板卡片后，把模板内容填入创意输入框
- 点击“AI 生成创意”后，用当前选中的小说类型自动填充创意
- 点击“生成大纲”后，调用接口并进入下一步
- 点击“取消”后，清空当前输入并返回工作台
```

### 5. 页面状态

```md
状态：
- 默认态：页面刚进入时显示什么
- 加载态：是否需要 skeleton / loading 文案 / 禁用按钮
- 空状态：未选择类型时模板区显示什么
- 错误态：字段错误显示在哪里
- 成功态：生成成功后跳转还是停留当前页
```

### 6. 接口说明

如果这个页面要接接口，请按下面格式写：

```md
接口：
- 接口名称：生成大纲
  地址：/api/xxx
  方法：POST
  入参：
    - genre
    - idea
    - platform
    - dna
    - wordCount
  成功后行为：跳转到 /dashboard/create/outline
  失败后行为：页面顶部显示错误提示
```

如果暂时不接接口，也可以直接写：

```md
接口：
- 暂不接后端，先做静态页面
```

### 7. 数据来源

```md
数据来源：
- 小说类型：前端写死 / 后端接口返回
- 模板数据：前端写死 / 后端接口返回
- 当前用户信息：GET /api/auth/session
```
